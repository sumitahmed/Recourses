Closure in Functional Dependencies

The closure of a set of attributes X, denoted as X+, is the set of all attributes that can be functionally determined from X based on a given set of functional dependencies (FDs). Attribute closure is used to test properties like dependency preservation and lossless decomposition.

Importance of Closure

  • Dependency Testing: To check if a dependency A → B is valid.
  • Key Identification: To determine if a set of attributes forms a candidate key.
  • Normalization: To simplify functional dependency sets during schema normalization.

Real-Life Relation

In a student database, the closure of StudentID includes Name, Department, and other attributes that can be determined from StudentID using functional dependencies.

Methods to Compute Closure of a Set

Iterative Application of Armstrong's Axioms

Armstrong's Axioms (Reflexivity, Augmentation, and Transitivity) are applied repeatedly to derive all attributes determined by the given set.

Problem

Given Functional Dependencies:

  • A → B
  • B → C
  • C → D
  • A → E

Find the closure of A (A+).

Step-by-Step Solution

  1. Start: Start with the given attribute set A: A+ = {A}.
  2. Iterate through the functional dependencies and add attributes determined by A+:
    • A → B: Since A ∈ A+, add B to A+.
      A+ = {A, B}
    • B → C: Since B ∈ A+, add C to A+.
      A+ = {A, B, C}
    • C → D: Since C ∈ A+, add D to A+.
      A+ = {A, B, C, D}
    • A → E: Since A ∈ A+, add E to A+.
      A+ = {A, B, C, D, E}
  3. Stop: When no new attributes can be added.

Solution

The closure of A, denoted as A+, is:

A+ = {A, B, C, D, E}

Conclusion

Computing the closure of an attribute set helps in understanding its functional dependencies and the role it plays in the schema. By iteratively applying Armstrong's Axioms, we can derive all attributes determined by a given set.
