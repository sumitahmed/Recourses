Advanced SQL Joins

Advanced JOINS allow for more sophisticated data combinations beyond basic matching, enabling complex queries for scenarios such as self-referencing, Cartesian products, and non-standard conditions.

1. SELF JOIN

A SELF JOIN is a join of a table with itself. It’s useful when you need to compare rows within the same table or retrieve hierarchical data.

Syntax:
SELECT a.column_name, b.column_name
FROM table_name a, table_name b
WHERE condition;

Example

Consider an Employee table:

Employee_ID Name Manager_ID
1 Alice NULL
2 Bob 1
3 Charlie 2
4 Dave 1

To retrieve each employee with their manager’s name:

SELECT e1.Name AS Employee, e2.Name AS Manager
FROM Employee e1
LEFT JOIN Employee e2 ON e1.Manager_ID = e2.Employee_ID;

Result:

Employee Manager
Alice NULL
Bob Alice
Charlie Bob
Dave Alice

2. CROSS JOIN

A CROSS JOIN returns the Cartesian product of two tables, meaning each row in the first table is paired with every row in the second table.

Syntax:
SELECT * FROM table1 CROSS JOIN table2;

Example

Consider Employee and Department tables:

Employee_ID Name
1 Alice
2 Bob
Department_ID Department_Name
101 HR
102 IT

Performing a CROSS JOIN:

SELECT Employee.Name, Department.Department_Name
FROM Employee
CROSS JOIN Department;

Result:

Name Department_Name
Alice HR
Alice IT
Bob HR
Bob IT

3. NATURAL JOIN

A NATURAL JOIN automatically joins two tables based on columns with the same name and compatible data types in both tables.

Syntax:
SELECT * FROM table1 NATURAL JOIN table2;

Example

Consider Student and Enrollment tables with a common column, Student_ID:

Student_ID Name
1 Alice
2 Bob
Student_ID Course_ID
1 CS101
2 MA101

Using NATURAL JOIN:

SELECT * FROM Student NATURAL JOIN Enrollment;

Result:

Student_ID Name Course_ID
1 Alice CS101
2 Bob MA101

4. EQUI JOIN

An EQUI JOIN combines rows based on matching values in specified columns, using an equality operator (=).

Syntax:
SELECT * FROM table1, table2
WHERE table1.column_name = table2.column_name;

Example

Consider Employee and Department tables:

Employee_ID Name Department_ID
1 Alice 101
2 Bob 102
Department_ID Department_Name
101 HR
102 IT

Using EQUI JOIN:

SELECT Employee.Name, Department.Department_Name
FROM Employee, Department
WHERE Employee.Department_ID = Department.Department_ID;

Result:

Name Department_Name
Alice HR
Bob IT

5. NON-EQUI JOIN

A NON-EQUI JOIN joins tables based on a condition other than equality, using operators like <, >, <=, or >=.

Syntax:
SELECT * FROM table1, table2
WHERE table1.column_name > table2.column_name;

Example

Consider Product and Price_Range tables:

Product_ID Product_Name Price
1 Laptop 1200
2 Smartphone 700
3 Mouse 20
Range_ID Min_Price Max_Price Range_Description
1 0 500 Low
2 501 1000 Medium
3 1001 2000 High

To find the price range for each product:

SELECT Product.Product_Name, Price_Range.Range_Description
FROM Product
JOIN Price_Range
ON Product.Price BETWEEN Price_Range.Min_Price AND Price_Range.Max_Price;

Result:

Product_Name Range_Description
Laptop High
Smartphone Medium
Mouse Low

Summary Table of Advanced JOIN Types

JOIN Type Description Example Outcome
SELF JOIN Joins a table with itself. Shows employees along with their managers’ names from the same table.
CROSS JOIN Returns Cartesian product of two tables. Each employee paired with each department.
NATURAL JOIN Joins tables based on columns with matching names. Combines Student and Enrollment on Student_ID without specifying join conditions.
EQUI JOIN Joins based on matching values in specified columns. Matches Employee and Department tables by Department_ID.
NON-EQUI JOIN Joins based on conditions other than equality. Matches Product with Price_Range based on the price falling within a specified range.
