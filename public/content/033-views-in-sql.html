Views in SQL

In SQL, a View is a virtual table based on the result set of a SQL query. It doesn’t store data itself but retrieves data from one or more underlying tables each time it’s accessed. Views allow users to encapsulate complex queries, filter sensitive information, and simplify data retrieval.

Key Points of Views

  • Virtual Tables: Views are stored queries that act as virtual tables.
  • Dynamic Data: Views pull data from underlying tables, always reflecting the latest information.
  • Read-Only or Updatable: Depending on the database and complexity, views can be either read-only or allow data modification.
Syntax:
CREATE VIEW view_name AS
SELECT columns FROM table_name WHERE condition;

Views vs. Tables

Feature View Table
Storage Does not store data itself; dynamically retrieves from tables Physically stores data
Data Dynamic; always reflects changes in the underlying tables Static unless updated directly
Creation Created with CREATE VIEW statement Created with CREATE TABLE statement
Modification Generally read-only (modifiable under certain conditions) Fully modifiable
Use Case Provides customized views of data without duplication Stores raw, actual data

Example

Consider a Sales table:

Sale_ID Product_ID Quantity Amount
1 101 2 200
2 102 1 150

To create a view showing only Product_ID and total Amount:

CREATE VIEW Sales_Summary AS
SELECT Product_ID, SUM(Amount) AS Total_Amount
FROM Sales
GROUP BY Product_ID;

Result:

Product_ID Total_Amount
101 200
102 150

Security and Views

Views provide a layer of data security by restricting access to specific columns or rows in a table. Sensitive data can be hidden by creating a view that excludes confidential columns.

Example

Consider an Employee table:

Employee_ID Name Department Salary
1 Alice HR 60000
2 Bob IT 80000

To create a view hiding the salary column:

CREATE VIEW Employee_View AS
SELECT Employee_ID, Name, Department
FROM Employee;

Result:

Employee_ID Name Department
1 Alice HR
2 Bob IT

Types of Views

Simple Views

Simple views are based on a single table and generally don’t use complex operations like joins or aggregations.

CREATE VIEW Employee_Info AS
SELECT Name, Department
FROM Employee;

This view simplifies access by presenting only the Name and Department columns.

Complex Views

Complex views are based on multiple tables and may include joins, aggregations, or subqueries.

CREATE VIEW Employee_Department AS
SELECT Employee.Name, Department.Department_Name
FROM Employee
JOIN Department ON Employee.Department_ID = Department.Department_ID;

Result:

Name Department_Name
Alice HR
Bob IT

Modifying Views

Views can be modified using the CREATE OR REPLACE VIEW statement without affecting permissions or dependent objects.

CREATE OR REPLACE VIEW view_name AS
SELECT columns FROM table_name WHERE condition;

Example

To add the Employee_ID column to the Employee_Info view:

CREATE OR REPLACE VIEW Employee_Info AS
SELECT Employee_ID, Name, Department
FROM Employee;

Result:

Employee_ID Name Department
1 Alice HR
2 Bob IT

Summary Table of SQL View Types and Usage

View Type Description Example Use Case
Simple View Based on a single table without complex operations. Hides columns in the original table for simplicity.
Complex View Based on multiple tables or includes operations like JOINs and aggregations. Combines data from multiple tables for comprehensive results.
