Types of SQL Subqueries

Subqueries in SQL can be categorized based on the type of results they return or their interaction with the main query. These categories include:

  • Single-row Subqueries
  • Multiple-row Subqueries
  • Multiple-column Subqueries
  • Correlated Subqueries
  • Non-Correlated Subqueries

1. Single-row Subqueries

Single-row subqueries return only one row as output and are often used with comparison operators such as =, <, >, <=, >=, or <>.

Example

Consider the Employee table:

Employee_ID Name Department Salary
1 Alice HR 60000
2 Bob IT 80000
3 Charlie HR 55000 
SELECT Name
FROM Employee
WHERE Salary = (SELECT MAX(Salary) FROM Employee);

Result:

Name
Bob

The subquery retrieves the maximum salary, and the main query selects the employee with that salary.

2. Multiple-row Subqueries

Multiple-row subqueries return more than one row and are typically used with operators like IN, ANY, or ALL.

Example

SELECT Name
FROM Employee
WHERE Department IN (SELECT Department 
                     FROM Employee 
                     GROUP BY Department 
                     HAVING COUNT(*) > 1);

Result:

Name
Alice
Charlie

The subquery retrieves departments with more than one employee, and the main query lists employees in those departments.

3. Multiple-column Subqueries

Multiple-column subqueries return multiple columns as output, often used in complex conditions requiring a combination of values.

Example

Consider the Product and Sales tables:

Product Table
Product_ID Product_Name Price
1 Laptop 1200
2 Smartphone 800
3 Mouse 20
Sales Table
Sale_ID Product_ID Quantity
1 1 5
2 2 10
3 1 2 
SELECT Product_Name, 
       Price * (SELECT SUM(Quantity) 
                FROM Sales 
                WHERE Sales.Product_ID = Product.Product_ID) AS Total_Sales
FROM Product;

Result:

Product_Name Total_Sales
Laptop 8400
Smartphone 8000
Mouse NULL

4. Correlated vs. Non-Correlated Subqueries

Subqueries can be divided into correlated and non-correlated subqueries:

Type Description
Correlated Subquery Refers to columns in the main query and is evaluated row-by-row.
Non-Correlated Subquery Runs independently of the main query and is executed only once.

Example: Correlated Subquery

SELECT Name
FROM Employee e1
WHERE Salary > (SELECT AVG(Salary) 
                FROM Employee e2 
                WHERE e1.Department = e2.Department);

Result:

Name
Bob

The subquery calculates the average salary for each department, and the main query returns employees earning above their department's average.

Example: Non-Correlated Subquery

SELECT Name
FROM Employee
WHERE Department = (SELECT Department 
                    FROM Employee 
                    GROUP BY Department 
                    ORDER BY AVG(Salary) DESC 
                    LIMIT 1);

Result:

Name
Bob
