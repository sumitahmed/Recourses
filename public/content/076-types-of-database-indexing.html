Types of Indexing in Databases

Indexing is a database optimization technique used to enhance query performance. Different types of indexing are tailored to address specific data storage and retrieval needs.

Types of Indexing

1. Primary Index

A Primary Index is created on the primary key of a table. It ensures that each row in the table is uniquely identifiable and organizes the table data physically in the order of the primary key.

Features:

  • One index per table.
  • Automatically created when a primary key is defined.

Example Table: Employee

EmpID Name Department
101 Alice HR
102 Bob IT
103 Charlie Finance

Index:

EmpID Pointer
101 Row 1
102 Row 2
103 Row 3

2. Secondary Index

A Secondary Index is created on columns other than the primary key to enhance query performance for specific use cases.

Features:

  • Supports non-primary key lookups.
  • Slower than primary indexes for retrieval.

Example:

Query by Department in the above table:

Department Pointer
HR Row 1
IT Row 2
Finance Row 3

3. Unique Index

A Unique Index ensures that all values in the indexed column are unique. This is commonly used for columns like email addresses or phone numbers.

Features:

  • Prevents duplicate values.
  • Automatically created for unique constraints.

Example:

A Unique Index on an email column in a customer table prevents duplicate email entries.

4. Composite Index

A Composite Index is created on two or more columns. It is useful for queries that involve filtering or sorting based on multiple columns.

Features:

  • Improves performance for multi-column queries.
  • Order of columns in the index matters.

Example:

Composite Index on Name and Department:

Name, Department Pointer
Alice, HR Row 1
Bob, IT Row 2
Charlie, Finance Row 3

5. Full-Text Index

A Full-Text Index is used for efficient text-based searches. It breaks down textual content into searchable keywords.

Features:

  • Ideal for blog or article databases.
  • Allows advanced searches like boolean or proximity queries.

Example:

Searching for "employee benefits" in a document repository.

6. Spatial Index

A Spatial Index is used to handle geographic and geometric data, such as locations or polygons.

Features:

  • Optimized for spatial data types (e.g., POINT, LINESTRING).
  • Used in Geographic Information Systems (GIS).

Example:

Querying a map for points within a radius of a given location.

7. Bitmap Index

A Bitmap Index represents data as bitmaps, which are arrays of binary values.

Features:

  • Efficient for low-cardinality columns (e.g., Gender, Status).
  • Performs well for analytic queries.

Example:

A Bitmap Index on a Gender column:

Row Male Female
1 1 0
2 0 1
3 1 0

Advantages and Disadvantages of Different Indexes

Index Type Advantages Disadvantages
Primary Index Guarantees uniqueness and order. Limited to one per table.
Secondary Index Enhances performance for non-key queries. Slower than primary indexes.
Unique Index Ensures data integrity. Overhead during insertions.
Composite Index Optimized for multi-column queries. Order of columns matters.
Full-Text Index Enables fast text searches. Storage-intensive.
Spatial Index Optimized for geographic data queries. Complex to maintain.
Bitmap Index Efficient for low-cardinality data. Inefficient for high-cardinality data.

Conclusion

Database indexing provides diverse tools to enhance query performance. Choosing the right type—Primary, Secondary, Composite, or specialized indexes like Full-Text and Spatial—depends on the nature of the data and query requirements.
