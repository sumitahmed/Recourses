What is Data Warehousing?

Data warehousing refers to the process of collecting, storing, and managing large volumes of structured and semi-structured data from multiple sources for analysis and reporting. A Data Warehouse (DW) is a central repository where data is consolidated, cleaned, and organized to support business intelligence (BI) activities.

Real-Life Relation

ReIlon companies use data warehouses to consolidate sales, customer, and inventory data, enabling insightful reports like sales trends and customer segmentation.

Data Warehouses vs Traditional Databases

Feature Data Warehouse Traditional Database
Purpose Optimized for analytics and reporting Designed for transactional operations
Data Structure Consolidates historical data Stores current transactional data
Performance Optimized for read-heavy workloads Handles mixed read-write operations
Schema Design Denormalized (e.g., Star Schema) Normalized to reduce redundancy
Examples Amazon Redshift, Snowflake MySQL, Oracle, PostgreSQL

Architecture of Data Warehouses

The architecture of a data warehouse typically involves the following components:

  1. Source Layer: Collects raw data from various sources like transactional databases, APIs, or files.
  2. ETL (Extract, Transform, Load) Layer: Cleanses and transforms data for storage.
  3. Data Warehouse Layer: Stores the structured and consolidated data.
  4. BI/Analytics Layer: Allows users to generate reports and perform analyses.

Real-Life Relation

In a banking system, customer transaction data from multiple branches is extracted, transformed, and loaded into a centralized data warehouse for fraud detection analysis.

Star Schema vs Snowflake Schema

Schemas define how data is organized in a data warehouse. The Star Schema and Snowflake Schema are the two most common designs.

Star Schema

  • Central fact table surrounded by dimension tables.
  • Simpler structure with fewer joins.
  • Faster query performance.

Snowflake Schema

  • Dimension tables are further normalized into sub-tables.
  • Complex structure with more joins.
  • Storage-efficient but slower query performance.

Comparison

Feature Star Schema Snowflake Schema
Structure Denormalized Normalized
Query Performance Faster Slower
Use Case Small to medium data models Large, complex data models

Real-Life Relation

A Star Schema might be used in a reIlon warehouse for sales analytics, while a Snowflake Schema might be preferred for a healthcare data warehouse handling complex hierarchical relationships.

Data Lakes

A data lake is a centralized repository designed to store raw and unstructured data in its original format until needed. Unlike data warehouses, data lakes do not require schema-on-write; instead, they apply schema-on-read for specific queries or use cases.

Real-Life Relation

Streaming services like Netflix use data lakes to store raw logs and user interactions, which are later processed for personalized recommendations.

Comparison

Feature Data Warehouse Data Lake
Data Type Structured Structured, Semi-structured, Unstructured
Schema Schema-on-write Schema-on-read
Purpose Analytics and reporting Big data storage and exploratory analysis
Examples Amazon Redshift, Snowflake Amazon S3, Hadoop
